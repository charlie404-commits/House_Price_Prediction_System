## 📊 Dataset

The notebook expects a `Housing.csv` file with these columns:

`price, area, bedrooms, bathrooms, stories, mainroad, guestroom, basement, hotwaterheating, airconditioning, parking, prefarea, furnishingstatus`

Place `Housing.csv` in the same folder as the notebook (or in a `data/` subfolder).
**If the file is missing, the notebook automatically generates a synthetic dataset with the
same schema so it still runs end-to-end** — replace it with your real dataset for genuine results.

## 🖥️ Running Locally (VS Code / Jupyter Notebook)

### 1. Clone or download the project
```bash
git clone https://github.com/<your-username>/house-price-prediction.git
cd house-price-prediction
```

### 2. Create a virtual environment (recommended)
```bash
python -m venv venv

# Activate it
# Windows:
venv\Scripts\activate
# macOS/Linux:
source venv/bin/activate
```

### 3. Install dependencies
```bash
pip install -r requirements.txt
```

### 4. Add your dataset
Place `Housing.csv` in the project root folder.

### 5. Open and run the notebook
**In VS Code:**
- Install the "Jupyter" extension (Microsoft) if not already installed.
- Open `House_Price_Prediction.ipynb`.
- Select your Python interpreter/kernel (the `venv` created above).
- Click **Run All**.

**In classic Jupyter Notebook:**
```bash
jupyter notebook
```
Then open `House_Price_Prediction.ipynb` in the browser and select **Run → Run All Cells**.

This will generate `house_price_model.pkl` and `label_encoders.pkl` in the project folder.

## 🚀 Deploying the Streamlit App

### Run locally
Make sure `house_price_model.pkl` and `label_encoders.pkl` (generated by the notebook) are in
the same folder as `app.py`, then run:
```bash
streamlit run app.py
```
The app will open automatically at `http://localhost:8501`.

### Deploy to Streamlit Community Cloud (free)
1. Push your project to a public (or private) GitHub repository (see commands below).
2. Go to [https://share.streamlit.io](https://share.streamlit.io) and sign in with GitHub.
3. Click **New app**, select your repository, branch, and set the main file path to `app.py`.
4. Make sure `house_price_model.pkl`, `label_encoders.pkl`, and `requirements.txt` are included
   in the repository (remove `*.pkl` from `.gitignore` if you want to version the trained model).
5. Click **Deploy**. Streamlit Cloud will install dependencies from `requirements.txt` and launch
   the app automatically.

### Alternative deployment options
- **Render / Railway**: Add a `Procfile` with `web: streamlit run app.py --server.port=$PORT --server.address=0.0.0.0`.
- **Hugging Face Spaces**: Create a new Space with the "Streamlit" SDK and push this repo to it.
- **Docker**: Build a container using a `python:3.10-slim` base image, `pip install -r requirements.txt`, and `CMD ["streamlit", "run", "app.py"]`.

## 🩹 Common Error Troubleshooting

| Error | Cause | Fix |
|---|---|---|
| `ModuleNotFoundError: No module named 'xgboost'` | XGBoost not installed | Run `pip install xgboost`, or let the notebook's automatic fallback installer handle it |
| `FileNotFoundError: Housing.csv` | Dataset not in expected location | Place `Housing.csv` in the project root, or let the notebook use its synthetic-data fallback |
| `KeyError` when encoding categorical columns | Sample/input data contains a category not seen during training (e.g., typo in "yes"/"no") | Ensure input values exactly match the categories used during training (case-sensitive) |
| Streamlit app shows "Model files not found" | `house_price_model.pkl` / `label_encoders.pkl` missing | Run the notebook first to generate these files, then place them next to `app.py` |
| `AttributeError: 'XGBRegressor' object has no attribute ...` | Version mismatch between XGBoost used to train vs. load the model | Keep the same `xgboost` version in `requirements.txt` used during training and deployment |
| Plots not displaying in VS Code | Jupyter extension/kernel not selected correctly | Re-select the Python interpreter/kernel and restart the kernel |
| `ImportError: cannot import name 'files' from 'google.colab'` | Running outside Google Colab | Expected — the notebook safely skips this via `try/except` and is only relevant on Colab |
| Slow training / high memory usage | Large dataset with `RandomForestRegressor`/`XGBRegressor` | Reduce `n_estimators`, or subsample training data for experimentation |
| `streamlit: command not found` | Streamlit not installed or not on PATH | Run `pip install streamlit` inside your active virtual environment |
